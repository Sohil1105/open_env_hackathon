BUG-PERF-2server/app.pycall_llm() + evaluate_applicant() HIGH 10 concurrent /evaluate requests trigger 50 simultaneous LLM API calls. HF router rate-limits or times out 4 of 10. Root cause: no request queuing or concurrency limiting on /evaluate. B1.1, B1.2
BUG-STATIC-2server/app.pyreset_environment() ~L272-276 MEDIUM global_session is only cleared when task_id == TASK_ORDER[0]. POST /reset with empty body does NOT reset global_session — completed_stages, stage_scores, and applicant_profile can leak between sessions. A8 (masked), STATIC-2
BUG-STATIC-1server/app.pyLifecycleSession class ~L155-162 MEDIUM LifecycleSession declares mutable completed_stages: list = [] and stage_scores: dict = {} as class-level attributes. BUG-C5server/app.pyCORSMiddleware ~L142-148 MEDIUM allow_origins=["*"] combined with allow_credentials=True violates CORS spec. Browsers reject credentialed requests (cookies, auth headers) to wildcard origins. All credentialed cross-origin requests silently fail in browsers. C5.4
BUG-STATIC-5server/app.pyApplicantInput.applicant_name ~L188 MEDIUM applicant_name has no min_length constraint. Empty strings ("") are accepted, processed, and passed to LLM prompt, resulting in a nonsensical applicant named "". Should return 422. A4.3
BUG-PERF-3server/app.pyuvicorn config MEDIUM Under concurrent evaluate load, /reset returns connection errors. Root cause: single-worker uvicorn + blocked event loop from synchronous LLM calls causes new connections to be refused when the connection queue is full. B4.2, B4.3
BUG-A2server/app.pyevaluate_applicant() + ApplicantInput HIGH annual_income=0 reaches grader with income=0. Despite grader guard dti = (debt/income) if income>0 else 1.0, the 500 fires due to ApplicantProfile validation (annual_income: float = Field(..., gt=0) in ApplicantProfile but not in ApplicantInput). A2.2
